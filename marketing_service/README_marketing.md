# Marketing Service

**Назначение:** Генерация идей контента с ИИ, ручное формирование плана публикаций, простой календарь «когда и что публиковать», ручная публикация в Telegram/VK, статусы: `draft → in_review → approved → (scheduled опционально) → published → archived`.

## Быстрый старт
1. Установи переменные окружения из `.env.example`.
2. Применяй миграции ядра и модуля (см. `migrations/`). Ядро не трогаем — только предметные таблицы.
3. Запуск сервиса:
   ```bash
   uvicorn marketing.app:app --reload
   ```
4. Префикс API: `/v1/marketing`.

## Группы API
- `posts` — CRUD постов, статусы, назначение даты, публикация в TG/VK, замена поста по дате.
- `sources` — источники (TG, URL/RSS), забрать «последние 10» материалов.
- `ai` — ИИ-генерация плана от промта; ИИ-идеи из импортированных материалов.
- `calendar` — очень простой срез «когда и что публиковать» + «корзина идей» (без дат).

## Связь с Core
- Workflow (переводы статусов), Audit (журнал действий), Docs (медиа/вложения к посту).
- `entity_type="mk_post"`, `entity_id=str(post_id)`.

## Что важно
- Дату публикации всегда проставляет пользователь.
- Календарь — только визуализация.
- Публикация — вручную по кнопке «сейчас» в TG/VK (расписаний нет).
